# Reverse Proxy
Từ mô hình LEMP + wordpress + laravel trước đó để xây dựng mô hình reverse proxy (nginx + apache)
Tại sao Nginx đứng trước Apache?

## Mô hình này tận dụng ưu điểm của cả hai web server:
- Nginx (Frontend - Reverse Proxy): * Ưu điểm: Kiến trúc hướng sự kiện (Event-driven), xử lý hàng chục nghìn kết nối đồng thời với lượng RAM cực thấp. Rất mạnh trong việc phục vụ file tĩnh (CSS, JS, hình ảnh) và xử lý SSL (SSL Offloading).
	+ Nhiệm vụ: Tiếp nhận yêu cầu từ người dùng, lọc traffic, xử lý SSL, sau đó chuyển yêu cầu xử lý file động (PHP) về cho Apache.
- Apache (Backend):
	+ Ưu điểm: Xử lý file động (PHP) ổn định với module mod_php hoặc php-fpm. Hỗ trợ file .htaccess linh hoạt cho từng thư mục (phù hợp với các mã nguồn như WordPress).
	+ Nhiệm vụ: Nhận yêu cầu từ Nginx, thực thi mã nguồn PHP và trả kết quả về cho Nginx.
- Nhược điểm khi triển khai mô hình: Cấu hình phức tạp hơn mô hình đơn lẻ, cần quản lý đồng bộ hai dịch vụ cùng lúc.

Giai đoạn 1: Cài đặt và Kích hoạt (1 điểm lưu ý nhỏ, nếu không phải user root thì thêm `sudo` ở đầu lệnh để chạy với quyền root)
1. Cài đặt dịch vụ
	```
	# cập nhật hệ thống 
	apt update
	# tải đầy đủ các gói cần thiết cho mô hình
	apt install nginx apache2 certbot python3-certbot-nginx python3-certbot-apache mysql-server php -y
	```
2. Kích hoạt Module cần thiết cho Apache
    - Apache cần các "tiện ích mở rộng" này để hiểu được SSL và các luật điều hướng của WordPress.
	```
	a2enmod ssl proxy proxy_http rewrite headers
	```
	+ ssl: Cho phép Apache chạy HTTPS (cổng 8443).
	+ proxy_http: Cho phép Apache nhận yêu cầu được chuyển tiếp từ Nginx.
	+ rewrite: Cực kỳ quan trọng để WordPress chạy được trang con (Permalinks).

Giai đoạn 2: Cấu hình Apache Backend (Port 8080/8443)
1. Cấu hình cổng lắng nghe
	- Tệp tin: `/etc/apache2/ports.conf`
	- Tại sao dùng: Thay đổi cổng tiêu chuẩn (80/443) sang cổng phụ vì Nginx đã sử dụng 2 cổng này.
	```
	Listen 8080
	Listen 8443
	```
2. Cấu hình VirtualHost chung
	- Tệp tin: `/etc/apache2/sites-available/site_vhost.conf`
	- Gom chung các site vào một file để dễ rà soát logic ưu tiên.
	```
	# Default Vhost (Chặn đứng mọi truy cập IP vào Apache)
	<VirtualHost *:8080 *:8443>
	    ServerName default
	    DocumentRoot /var/www/giahung
	    <Location />
	        Require all denied
	    </Location>
	</VirtualHost>
	
	# Vhost WordPress
	<VirtualHost *:8080 *:8443>
	    ServerName wp.giahung.vietnix.tech
	    DocumentRoot /var/www/giahung/wordpress
	    
	    <Directory /var/www/giahung/wordpress>
	        AllowOverride All
	        Require all granted
	    </Directory>
	
	    # Kích hoạt SSL cho cổng 8443
	    SSLEngine on
	    SSLCertificateFile /etc/letsencrypt/live/wp.giahung.vietnix.tech/fullchain.pem
	    SSLCertificateKeyFile /etc/letsencrypt/live/wp.giahung.vietnix.tech/privkey.pem

		# Tạo log để có thể theo dõi thời gian thực 
	    ErrorLog ${APACHE_LOG_DIR}/wordpress_error.log
	    CustomLog ${APACHE_LOG_DIR}/wordpress_access.log combined
	</VirtualHost>
	
	# Vhost Laravel
	<VirtualHost *:8080 *:8443>
	    ServerName laravel.giahung.vietnix.tech
	    DocumentRoot /var/www/giahung/laravel/public
	    
	    <Directory /var/www/giahung/laravel/public>
	        AllowOverride All
	        Require all granted
	    </Directory>
	
	    SSLEngine on
	    SSLCertificateFile /etc/letsencrypt/live/laravel.giahung.vietnix.tech/fullchain.pem
	    SSLCertificateKeyFile /etc/letsencrypt/live/laravel.giahung.vietnix.tech/privkey.pem
	
	    ErrorLog ${APACHE_LOG_DIR}/laravel_error.log
	    CustomLog ${APACHE_LOG_DIR}/laravel_access.log combined
	</VirtualHost>
 	```

Giai đoạn 3: Cấu hình Nginx Frontend (Port 80/443)
1. Tối ưu bảng băm Header
  + Tệp tin: `/etc/nginx/nginx.conf`
  + Tại sao dùng: Tránh lỗi cảnh báo could not build optimal proxy_headers_hash khi hệ thống phải xử lý quá nhiều Header truyền tin.
Nginx
	```
  	# thêm vào block http 2 dòng dưới
	http {
	    proxy_headers_hash_max_size 512;
	    proxy_headers_hash_bucket_size 128;
	    ...
	}
	```

2. Cấu hình Reverse Proxy & Static Offloading
  + File: `/etc/nginx/sites-available/wordpress`
Nginx
	```
	server {
	    listen 443 ssl;
	    server_name wp.giahung.vietnix.tech;
	
		ssl_certificate /etc/letsencrypt/live/wp.giahung.vietnix.tech/fullchain.pem;
	    ssl_certificate_key /etc/letsencrypt/live/wp.giahung.vietnix.tech/privkey.pem;
	
	    # Nginx tự phục vụ ảnh/css/js
	    location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff|woff2|ttf|svg|scss)$ {
	        root /var/www/giahung/wordpress;
	        expires 30d;
	        add_header X-Static-Handler "Nginx-Handled-This-File";
	    }
	
	    # Reverse Proxy: Đẩy PHP sang Apache
	    location / {
	        proxy_pass https://127.0.0.1:8443;
	        proxy_http_version 1.1;
	
	        # Bộ thông số SNI (Quan trọng để fix lỗi 421)
	        proxy_ssl_server_name on;
	        proxy_ssl_name $host;
	        proxy_ssl_verify off;
	
	        # Truyền Header định danh khách hàng (user)
	        proxy_set_header Host $host;
	        proxy_set_header X-Real-IP $remote_addr;
	        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
	        proxy_set_header X-Forwarded-Proto $scheme;
	    }
	}
	```
+ File: `/etc/nginx/sites-available/wordpress`

	```
	server {
	    listen 80;
	    server_name laravel.giahung.vietnix.tech;
	    return 301 https://$host$request_uri; # Ép buộc dùng HTTPS để bảo mật
	}
	
	server {
	    listen 443 ssl;
	    server_name laravel.giahung.vietnix.tech;
	
	    # CỰC KỲ QUAN TRỌNG: Phải dùng đúng Cert của Laravel
	    ssl_certificate /etc/letsencrypt/live/laravel.giahung.vietnix.tech/fullchain.pem;
	    ssl_certificate_key /etc/letsencrypt/live/laravel.giahung.vietnix.tech/privkey.pem;
	
	    # Root của Laravel phải trỏ vào thư mục /public
	    root /var/www/giahung/laravel/public;
	    index index.php index.html index.htm;
	
	    # 1. Static Offloading (Nginx tự xử lý file tĩnh của Laravel)
	    location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff|woff2|ttf|svg|scss)$ {
	        # Đảm bảo root ở đây cũng trỏ đúng
	        root /var/www/giahung/laravel/public;
	        expires 30d;
	        add_header X-Static-Handler "Nginx-Handled-This-File";
	        access_log off;
	    }
	
	    # 2. Dynamic Content (Proxy sang Apache 8443)
	    location / {
	        proxy_pass https://127.0.0.1:8443;
	        proxy_http_version 1.1;
	
	        # Giữ nguyên bộ SNI để tránh lỗi 421/400
	        proxy_ssl_server_name on;
	        proxy_ssl_name $host;
	        proxy_ssl_verify off;
	
	        # Header chuẩn cho Laravel nhận diện đúng Client IP
	        proxy_set_header Host $host;
	        proxy_set_header X-Real-IP $remote_addr;
	        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
	        proxy_set_header X-Forwarded-Proto $scheme;
	    }
	}
	```
Giai đoạn 4: Bảo mật và Ứng dụng (Security & App)

Nginx
	```
	server {
	    listen 80 default_server;
	    listen 443 ssl default_server;
	    server_name _;
	    # Sử dụng cert bất kỳ để tránh lỗi khởi động
	    ssl_certificate /etc/letsencrypt/live/wp.giahung.vietnix.tech/fullchain.pem;
	    ssl_certificate_key /etc/letsencrypt/live/wp.giahung.vietnix.tech/privkey.pem;
	    return 403;
	}
	```

## Với cấu hình hiện tại http sẽ redirect sang https
Chỉnh sửa cấu hình để chạy song song http và https
- File: `/etc/nginx/sites-available/wordpress`, cập nhật cấu hình trong cụm `80`
	```
	server {
    listen 80;
    server_name laravel.giahung.vietnix.tech;
    root /var/www/giahung/laravel/public;

    # Xử lý file tĩnh cho HTTP
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff|woff2|ttf|svg|scss)$ {
        expires 30d;
        add_header X-Static-Handler "Nginx-Handled-This-File-HTTP";
    }

    # Đẩy sang Apache cổng 8080 (Không mã hóa)
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme; # Sẽ truyền là "http"
    }
 	# bỏ dấu '#' để http direct sang https
	#    return 301 https://$host$request_uri; # Force HTTPS
	}
 	```
 
- File: `/etc/nginx/sites-available/laravel`, tương tự cập nhật cấu hình trong cụm 80
	```
	server {
    listen 80;

        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme; # Sẽ truyền là "http"

    # bỏ dấu '#' để http direct sang https
	#    return 301 https://$host$request_uri; # Force HTTPS

- File: `/var/www/giahung/wordpress/wp-config.php`, thêm các dòng đường dẫn vào đầu file
	```
	if (isset($_SERVER['HTTP_X_FORWARDED_PROTO']) && $_SERVER['HTTP_X_FORWARDED_PROTO'] == >

	}
	// Tự động nhận diện giao thức để tránh WordPress tự redirect
